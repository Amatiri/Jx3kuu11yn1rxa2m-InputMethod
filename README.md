# 解书音形编码规则

## 前言

解书音形是一种**音形结合、零重码、可自定义**的现代汉字输入方案，采用ABCD(E)(.F)的编码结构，兼顾**易学性**和**高效率**。本输入法秉承“**知音辨形，高效输入**”的理念，让用户在掌握拼音的基础上，通过少量形码规则即可实现高速盲打。解书音形的编码是一个充分考虑字源字义，又以现代字型为准的编码方案，保持文化敏感：编码反映汉字的结构演变和文化内涵，以音切入，以义定形，动态分配，注释避重。

## 一、编码结构概览

每个汉字的完整编码由以下部分组成：

```
ABCD(E)(.F)
```

其中：

- **ABC（音码）**：基础音码，忠实记录真实、准确的发音（必填）
- **DEF（形码）**：书写描述。有一个重要的原则：在规范、直觉、效率之间做平衡
  - **D**：主码：部首/独体区分形码（必填）
  - **E**：副码：余部/查重注释形码（可选）
  - **F**：补码：繁异/同义补充形码（可选）

**核心要点：**

- **先音后形**：前三位ABC是**拼音**，后几位DEF是**字形**
- **常用优先**：最常用字编码最短
- **零重码**：每个编码都有唯一汉字

**音码：** 搜狗双拼+声调，0轻声。

**主码：**

1. **独体数字编码（按优先级）**
   3开形(八) > 6大形(大) > 0闭形(口) > 5称形(中) > 1竖形(千)> 2横形(一) > 4错形(互) > 7偏形(厂) > 9复形(重) > 8奇形(乂)

2. **合体字母编码（部首→部首码）**
   
   - **关键部首码**：
     - r=人/亻，k=口，m=木，s=示/礻，t=土
     - u=氵(水)，x=心/忄，y=言/讠，z=辶
   - 完整部首表作为参考

**副码：**

1. **何时需要副码？**
   
   - 同音码+同部首码的字（前四码都相同）（ABCD四位编码为“前四码”）

2. **副码选择顺序**
   空 > 部首别码 > 余字解析码 > e > ; > 其他字母

**补码与特殊规则**

1. **补码格式：.X**
   
   - 主要用于繁体字、异体字
   - 日常输入很少用到

2. **特殊规则摘要**
   
   - 独体字没有副码
   - 最常用字没有副码
   - 未简字副码固定为;
   - 余单笔画有特殊编码

**快速入门路径：** 第1步：掌握音码。第2步：掌握主码基础。第3步：常用字实战练习。第4步：逐步学习副码。

**设计理念总结**

1. **三层结构**
   音码(ABC) → 部首(D) → 细化(E/F)
   
   - 知道读音就能输入
   - 记少量形码就能精确

2. **渐进学习**
   
   - 初期：只记前四码
   - 中期：加副码区分重码
   - 高级：掌握全规则

3. **实用主义**
   
   - 规则为效率服务
   - 相似字编码模式一致
   - 最常用字编码最短

**学习建议**：**先用再学**、**实战为主**、**善用工具**。

这套输入法在**易学性**和**效率**之间取得了良好平衡：知道拼音就能打，记住少量形码就能快，适合追求效率又不愿死记硬背的用户。

**初码（A）声母：**\
zh，ch，sh→v，i，u。

其他声母同全拼。

**韵码（B）韵母：**

## 三、主码编码规则

主码是编码中的第四位，用于区分同音字。它根据汉字的字形结构，将汉字分为**独体字**和**合体字**两大类，并赋予不同的编码规则。因此，主码又称“形部码”。

### 1. 合体字判定标准

一个汉字是否属于合体字，按照以下标准判断：

- **可拆分性**：能够拆出**非单笔画部首**的字，视为合体字；否则为独体字。“拆”要求独立明显，不交错、不穿插、不粘连。取完部首后，剩下的部件应尽量成字，避免拆分出无意义的部件。
- **现代字形原则**：所有字源不清的字（包括错释、模棱两可、关系复杂、隶变等），统一按现代字形处理。隶变后变得模糊难以辨认的部件，一般跳过不取。
- **解书拆分法**：为了实现精确拆分，我们采用“解书”拆字法。

### 2. 独体字的分类与优先级（独体构形）

独体字根据其字形特征分为以下10类，并按照优先级从高到低排列（数字为主码）：

| 类别  | 主码  | 特征说明                                 | 例字    |
| --- | --- | ------------------------------------ | ----- |
| 开形  | 3   | 笔画相互分开                               | 二三六八火 |
| 大形  | 6   | 横、撇、捺构成汉字主要框架                        | 矢水米乘求 |
| 闭形  | 0   | 有占据主导地位的封闭部件                         | 自酉曲凸甘 |
| 称形  | 5   | 可构成上下或左右对称                           | 中且臣丞丽 |
| 竖形  | 1   | 字形瘦高，或有贯穿汉字的竖画                       | 生主已幺毛 |
| 横形  | 2   | 字形矮宽，或有贯穿汉字的横画                       | 人之而瓜也 |
| 错形  | 4   | 有占主导的交错粘连的笔画                         | 刀舟正不壬 |
| 偏形  | 7   | 汉字重心不稳，左下角或右下角空缺                     | 产勿今母耳 |
| 复形  | 9   | 结构复杂，包含多个符合上述形状的结构；或隶变后无法辨认拆形而导致的独体字 | 南先弗巳重 |
| 奇形  | 8   | 以上字形都不符合，或以上字形都被更常用的字占用所产生的重码独体字字形   | 乙乂    |

**注意**：

- 优先级表示当某个汉字符合多个类别时，按优先级高的类别编码。
- 主码为数字0-9，分别对应上述类别。对于独体字的衍生，只有当衍生出的新字并未构成任何新构形时，才采用原构形；否则均认为原构形被打破，使用新构形。
- 独体字现代字形必须像独体字——这是判断复形独体字的重要标准。

### 3. 合体字的部首选取规则

合体字的主码为其**部首**对应的字母（部首码），部首的选取遵循以下规则：（康熙部首+优化分配）

#### 3.1 基本原则

1. **直觉优先：** 按直觉如果能拆出一个明确无歧义的部首，则直接取该部首码。
2. **康熙部首验证**：如果按直觉拆出多个部首，或按直觉难以确定部首，则根据《康熙字典》所规定的部首确定。根据常识，部首优先为形旁。汉字的部首不能取自己（即不能用整个字作为自己的部首），但是可以取部首的变体。
3. **优化分配**：若康熙部首取自己，或者在现代字形中难以辨认；或者出于避重、历史编码因素考虑，需要进行部首的优化分配（3.3）。
4. **部首表对应**：大多数部首已在部首表中给出，直接对应其部首码（见部首表），如果不在部首表中，则需要进行处理（3.4）。

#### 3.3 优化分配方法

由于“现代字形，避重，历史”三个原因，部首的优化分配诞生了以下方法：

1. **独体字/合体字判定**：确保合体字判定准确。
2. **康熙部首为本字**：按解书拆分进一步取部首。
3. **主码冲突**：当不同部首对应同一部首码（字母）时，可以调整以避免重码。
4. **亲缘衍生忽略**：若音码不同，可忽略亲缘衍生关系。
5. **特定部首规则**：部分部首有特殊规则（如含彳取彳，含廾取廾）。
6. **现代字形与左上原则**：部首的选取需在现代汉字结构下具有区分度。如果传统部首因隶变、简化等原因在现代字形中难以辨认，或造成重码问题/与现代字形差异过大，则按解书拆分选取首部（左上角部件）。当依旧无法确定部首时，优先取左上角部件。
7. **包围声旁处理**：声旁特征不明显或被外部特征覆盖时，若仅此一字如此，则取外声旁；若多字如此，则取内部件。
8. **简体问题**：有些简化字按简化前的繁体字形处理。
9. **简并/非部首字合理跳过**：若有其他部首，则不取简并部首或者未收录的成字部首。
10. **字源参考**：在无法决断时，可参考字源。

尽管方法众多，但通常情况下，《康熙字典》所规定的部首就是最优解。

### 4. 主码编码总结

- **独体字**：主码为数字（0-9），根据字形特征按优先级选取。
- **合体字**：主码为字母（b-z，不含a、e、;），对应部首表中的部首。
- **禁用字母**：a、e、; 不用于主码。

## 四、副码编码规则

副码是编码中的第五位，为**可选扩展码**。其主要作用是在前四码相同的一组字内实现**零重码**。副码的设计体现了“余部集合、部首传递”的核心思想。

**副码的存在规则**

- **副码省略原则**：为缩短码长，最常用字**没有副码**（即空副码）。为统一码长，需要占位时使用a作为占空码，在码表中省略不写。
- **独体字规则**：独体字的副码一律为空，依靠主码数字区分实现零重码。
- **常用度排序**：在同一前四码组内，汉字按常用度排序：
  最常用字：空副码
  次常用字：优先占用高优先级字母
  后续字：若首选字母已被占用，则顺延至下一个可用字母

**常用度判定标准**

当两个汉字常用度差别不大时，按以下优先级判定：可读性 > 含义丰富度 > 形声字声旁匹配度 > 笔画数 > 起笔。

**术语定义**

| 术语         | 定义                                           |
| ---------- | -------------------------------------------- |
| **余集合/余字** | 汉字去除主码部首后剩余的部分，如果“余集合”成字，则称“余字”。             |
| **余音**     | 余字的初码（双拼声母）                                  |
| **余部**     | 余集合按“解书拆分”法进一步拆分的部件                          |
| **音似/音异**  | 余字与整字音码相似度：<br>• 宽音码高度相似（初码主导）为音似<br>• 否则为音异 |

**注意**：音似/音异判断以今音为主，参考古音。

“宽音码”指的是音码去掉调码后的初韵码组合。

**副码选择优先级**

副码的选择遵循严格的优先级顺序：

**空副码 > 部首别码 > 余字解析码（余音/余部） > e > ; > 其余字母**

### 1. 部首别码

**条件**：余集合本身是《部首表》中的**独立部首**，且其部首码与其初码（余音）不同。则其部首码亦称为部首别码。反之若部首码与初码相同，则称为无部首别码。对于有部首别码的部首，称为“别码部首”或者“另记部首”，需要着重记忆。

**例外**：

1. 非部首字（如夫→矢）不在部首表中，无部首别码。
2. 部首变体（如 龶→青）视作有部首别码。

### 2. 余字解析码

当无部首别码时，按以下逻辑选择：

| 情况           | 处理方式              |
| ------------ | ----------------- |
| **余集合音似或难读** | 优先取余部，可采用余部首传递    |
| **余字音异且易读**  | 优先取余音             |
| **独体字作余字**   | 若无“亠丷”等部件，则视作仅有余音 |

**递归嵌套规则**：\
当余集合内部仍可拆分时，允许递归嵌套：

- “音”作为余字 → 余部“立” → “立”作为新余字 → 余部“亠”，余音l
- “多”作为余字 → 余部“夕” → “夕”作为新余字 → 部首别码t，余音x

**余部首传递规则**

**1. 基本传递**

取余部时，如果余集合成字，则通常优先取余字的部首作为副码来源。若部首不在部首表中，可取部首余音。

**示例**：

- “溢”：部首“氵”（水）→ 主码u，副字“益”为合体字，主码为m
- “溢”的副码取m（“益”的主码）

**2. 结构特殊规则**

在上下（左右）结构的字中，若满足以下条件：

1. 下（右）部为部首
2. 上（左）部为音似余字
3. 余字为上下（左右）结构，下（右）部为部首

则取**余字上（左）部**作为副码，避免“抛外取内”。

**综合示例**：“意”字的编码分析

1. 部首分析：
   
   - “音”在康熙字典中为“音部0画”
   - 部首不能取自己，将“音”拆为“立”+“日”
   - “立”是简并部首（立→亠），“日”是正宗部首
   - 根据“简并部首跳过”原则，取“日”为部首 → 主码r

2. 余字分析：
   
   - “意”为上下结构：上“音”下“心”
   - 部首为“心”（下部），余字为“音”（上部）
   - “音”的余部为“立”+“日”，取上部“立”
   - “立”的余音（初码）为l

3. 副码确定：
   
   - “意”的副码取l（“立”的余音）

**现代字形原则**

余部的选取依然遵循**现代字形原则**：

- 所有字源不清的字（包括错释、模棱两可、关系复杂、隶变等），统一按现代字形处理
- 隶变后变得模糊难以辨认的部件，一般跳过不取
- 使用解书拆字法
- 当出现重码问题时，加上现代字形的考虑。

**特殊规则**

**1. 未简字处理**

在Unicode基本区内，若有一个字**未进行类推简化**，缺失对应简体，称为“未简字”。

**规则**：未简字的副码一律为;（分号）

**2. 余单笔画处理**

当一个字取完部首后**仅剩单笔画**，该笔画可取为副码。

**笔画与字母对应**：“丶一丨丿乙乛𠃌乚𡿨”分别对应“dhupygkwv”。

有：系xi4sp/ji4sp。

**宏观验证原则**

为确保编码系统的一致性和合理性，设置以下验证原则：

**1. 对称性原则**

**字形、结构相似的字，其编码模式应尽量保持一致。**

**2. 多条件优先**

**同时满足两个条件时，优先取支持原因多的条件。**

**3. 实用主义**

在规则冲突时，以实际编码效果和用户记忆负担为最终考量。

**4. 副码优化分配**

类似主码，副码也有适当优化分配。需要理解的字例有：进zw。

## 五、简化解书拆分规则

解书拆分规则是一套用于精确描述汉字结构的**递归式语法系统**。它通过结构化编码实现汉字的拆分，为输入法编码提供字形分析基础。解书拆分规则本身**不直接用于输入**，而是作为主码和副码选取的**理论依据和辅助工具**。

解书拆分规则采用一套简洁的符号系统来描述汉字结构：

| 类别    | 符号  | 含义           |
| ----- | --- | ------------ |
| 左右结构  | r.  | ⿰（左右并列）      |
| 左中右结构 | rr. | ⿲（左中右并列）     |
| 上下结构  | i.  | ⿱（上下叠加）      |
| 上中下结构 | ii. | ⿳（上中下叠加）     |
| 包围结构  | l.  | ⿸/⿹/⿺等（各种包围） |
| 交错结构  | x.  | ⿻（笔画交错，少用）   |

**部件表示法**

| 表示法      | 格式          | 说明                     | 示例               |
| -------- | ----------- | ---------------------- | ---------------- |
| 部首双码     | 部首码 + 部首次码  | 部首在部首表中的位置对应部首次码       | 贝 → bz           |
| 独体字码     | 前四码         | 独体字的完整四码表示             | 天 → tm16         |
| **扩展字根** | **两个字母的代码** | **非部首表中的必要字符（见扩展字根表）** | **立 → ld**       |
| 简化表示     | 见下          | 实际编码中使用的简化形式           | 贝 → bb，立 → dj/lh |

**注释语法**

| 符号    | 含义        |
| ----- | --------- |
| \(u\) | 取部件的上部    |
| \(d\) | 取部件的下部    |
| \(l\) | 取部件的左部    |
| \(r\) | 取部件的右部    |
| \(f\) | 繁体形式      |
| \(j\) | 简体形式      |
| \(b\) | 变体形式      |
| \(x\) | 去掉某部分（少用） |

**集合语法**

| 符号   | 含义                             |
| ---- | ------------------------------ |
| \[\] | 将相关部件分组，如[i. sj kb 4] 表示一个上下结构 |
| 々    | 重复前一部件                         |

**细调数字**

| 数字  | 含义   |
| --- | ---- |
| 1   | 左右微调 |
| 2   | 上下微调 |
| 3   | 包围微调 |
| 4   | 交错微调 |

**编码流程**

**步骤1：拆分**

按结构码递归拆分汉字至基础部件，直到所有部件都可用部首双码、独体字码或扩展字根表示。

**拆分原则**：

- 优先按明显结构拆分（左右→上下→包围）
- 包围结构统一用l.表示
- 允许编码简写（如fu4r替代重复“付”结构）

**步骤2：表示**

部件用以下方式表示：

1. **部首部件**：用部首双码表示（部首码+部首次码）
   - “心”是部首表中x的第三个部首，对应部首次表中x的第三个字母n，组合为xn
2. **独体字部件**：用前四码表示
3. **特殊部件**：用扩展字根表示

**步骤3：组合**

按结构码组合部件序列，用空格分隔各部件。

“对”：r. ky cp

- r.：左右结构
- ky：“又”的部首双码
- cp：“寸”的部首双码

**简化表示法（实际编码使用）**

在实际的输入法编码生成中，为简化处理，采用以下简化表示：

1. **部首双码简化：** 部首码 + b（作为部首提示）
2. **前四码简化：** 独体字：取初码 + d（独体标记）合体字作部件：取初码 + h（合体标记）
3. **扩展字根简化：**
   - 部首简并的：用简并部母 + j
   - 罕部跳过的：并入独体字表示
   - 其他扩展字根：用占位符号ee
   - 可进一步拆分的扩展字根：拆分表示
4. **笔画简化：** 根据副码的单笔画对应字母 + a

**核心原则：**

- **实用性优先**
  
  - 包围结构统一用l.表示，不区分具体包围类型
  - 允许编码简写，如用已有字的编码表示重复部件
  - 同一汉字可能有不同的拆分表示，以实际拆分效果为准，只要逻辑自洽、可拼回原字即为有效，不追求理论完美，不是刚性编码标准，允许人工调整

- **可读性关联**
  
  - 部首次码优先采用部件韵母对应的字母
  - 使编码具有一定提示性，便于人工识别
  - 如“白”→bl（韵母ai→l）与拼音关联

- **人工参考定位**
  
  - 解书拆分作为**字形分析框架**而非输入编码
  - 通过解书拆分识别“可再分、表意强”的部首，为合体字的主码选取提供依据
  - 提供余集合拆分的部件依据，帮助确定副码的余音或余部
  - 在编码冲突时提供讨论和验证的基础，为主/副码选取提供多个结构视角，帮助在多个可能编码中做出选择
  - 确保拆分可追溯、可讨论，贯彻“以义定形”理念：每个拆分步骤都有明确的结构依据，相似结构的字采用相似的拆分方式

**注意事项**

1. **人工判断优先**：当规则冲突时，以人工判断的合理拆分优先
2. **持续优化**：扩展字根表和规则可根据实际使用情况持续优化

解书拆分为解书音形提供了坚实的字形分析基础，使主码和副码的选取不再依赖于直觉，而是有章可循的系统工程。它体现了本输入法“形音义结合，理据性优先”的设计哲学。

## 六、补码编码规则

补码用于处理汉字在Unicode范围内的所有繁、异体形式，包括同义异写字、大写数字、通假古音字等，以前缀点.引导。补码一般用不到。\
通常，“.”直接表示该字在常用语境下的**第一繁体**（如“纪”之“紀”）。\
若某字仅有单一繁体或特殊异体，则使用复合后缀区分：.f（大陆繁体）、.g（港澳异体）、.t（台湾异体）、.j（日本汉字）、.u（通假/古音字）、.k（韩国用字）、.z（并字）等。\
若某字对应多个常用繁体（如“复”之“復”、“複”），则其补码的选取规则与副码相同（即类比为新的“余字”进行区分），如.t、.y。\
**全覆盖与零重码**：ABCDE.F形式的**解书码**可覆盖Unicode收录的所有汉字（不包括喃字等独立文字系统），并保持全局零重码。

## 词语建议

### 一、词语编码的核心原则

**1. 基本定义**

- **广义词语**：包括多字词、常用短语、自定义短语、输入法特征词、通用符号等
- **补充功能**：外置于“多字首选字”输入的辅助方法，旨在提高输入效率

**2. 硬性规则**

- **最小长度**：词语编码至少3位，且这三位不能包含数字。
- **零重码**：一个编码仅对应一个词语（但一个词语可对应多个编码）
- **长度原则**：常用词语分配短编码，非常用词语使用长编码

### 二、多字词编码建议

**1. 二字词**

最简码 = 第一字宽音码 + 第二字初码

扩展顺序：优先补充第二字的韵码（最常用） → 补充第二字的调码 → 补充第二字的主码

**2. 三字词**

最简码 = 第一字初码 + 第二字初码 + 第三字初码

扩展顺序：补第一字韵码 → 补第二字韵码（最常用） → 补第三字韵码 → 补第三字调/主码

**3. 四字词**

方案1（备用）：第一字初码 + 第二字初码 + 第四字初码

方案2（常用）：四个字各取初码

扩展顺序：补前三字韵码（常用） → 补第四字韵码 → 补第四字调/主码

**4. 五字及以上**

方案1：第一字初码 + 第二字初码 + 最后字初码

方案2（适应习惯，常用）：前三字初码 + 最后字初码

扩展顺序：补所有字初码（常用） → 补最后字外的韵码 → 补最后字编码

### 三、通用符号编码方案

**1. 编码结构**

类型符（1字母）+ 具体编码（至少2字母） = 3位最小码

**2. 类型符建议**

| 类型符 | 符号类型 | 示例      |
| --- | ---- | ------- |
| b   | 部首   | bdh → 亠 |
| r   | 日文假名 | rka → か |
| d   | 中文标点 | d..→。   |
| t   | 特殊符号 | tzo→“←” |
| g   | 希腊字母 | gal→α   |
| u   | 数学符号 | uqh→∑   |

**剩余符号待更新**

**3. 具体编码规则**

- **部首**：部首双码，如“亠”→ bdh（b部首码，d为“亠”在b部首次表第4位。）
- **假名**：罗马字转写，如“か”→ rka

### 四、空码区的利用策略

**1. 空码区类型**

- **拼音缺失区**：如“bb”（bou不存在）、“cc”等
- **声调缺失区**：调码5-9（实际只有0-4）

**2. 利用方案**

- **预留区**：“bb0”用于无拼音汉字（已在newedit中使用）
- **用户自定义区**：其他空码区开放给用户自由定义

**3. 自定义建议**

- **个性化汉字**：将个人常用汉字放入空码区
- **快捷输入**：设置更短汉字编码（如“yi8x”→“意”）
- **特殊双拼**：为不使用标准双拼的用户预留空间

## 码表挂接

使用无数字版码表，便于主流输入法系统识别。无数字四位码实现如下：

首先将五个音调改为两个平仄：12→a，340→e。

前面双拼两位基本不变，除了ing适配：b,d,p,t和y的ing为d。jqx的ing为a。lmn的ing为v。

独体字：先平仄，再字形。每个平仄又分两到三声，第一声（平声中的阴平，仄声中的上声）字形字母（0-9）为pqwertyuio。第二声（平声中的阳平，仄声中的去声）字形字母为masdfghjkl。第三声（即仄声中的轻声）字形字母为zxcvbncvbn。

合体字：

先部首码，再附加。附加分副码是否省略：省略副码：用平仄。副码不省略：用副码。
